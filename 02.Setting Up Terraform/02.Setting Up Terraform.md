# 2. Setting Up Terraform

Now that we understand **what Terraform is and why it's useful**, let's get our hands dirty by installing Terraform and writing our first configuration.

## **2.1 Installing Terraform**

Terraform is available for **Windows, Linux, and macOS**. The installation steps vary based on your operating system.

### **🔹 Installing on Linux (Ubuntu/Debian)**

```bash
$ sudo apt update && sudo apt install -y gnupg software-properties-common

$ wget -O- https://apt.releases.hashicorp.com/gpg | sudo gpg --dearmor -o /usr/share/keyrings/hashicorp-archive-keyring.gpg

$ echo "deb [signed-by=/usr/share/keyrings/hashicorp-archive-keyring.gpg] https://apt.releases.hashicorp.com $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/hashicorp.list

$ sudo apt update && sudo apt install -y terraform
```

✅ **Verify installation**:

```bash
$ terraform version
```

### **🔹 Installing on macOS (Homebrew)**

```bash
$ brew tap hashicorp/tap
$ brew install hashicorp/tap/terraform
$ terraform version
```

### **🔹 Installing on Windows**

1. Download Terraform from the official site: Terraform Downloads
2. Extract and add the binary to the **system PATH**.
3. Open **Command Prompt (cmd)** and verify installation:
    
    ```powershell
    $ terraform version
    ```
    

## **2.2 Configuring Terraform for Cloud Providers**

Terraform requires authentication to interact with cloud providers like **AWS, Azure, and GCP**.

### **🔹 Configuring AWS for Terraform**

1. **Install AWS CLI** (if not installed):
    
    ```bash
    $ sudo apt install awscli -y  # Ubuntu
    $ brew install awscli          # macOS
    ```
    
2. **Configure AWS Credentials**:It will ask for:
    
    ```bash
    $ aws configure
    ```
    
    - AWS Access Key
    - AWS Secret Key
    - Default region (e.g., `us-east-1`)
    - Output format (`json` recommended)

📌 These credentials will be stored in `~/.aws/credentials` (Linux/macOS) or `C:\Users\YourUser\.aws\credentials` (Windows).

### **🔹 Configuring Terraform for Azure**

1. Install the Azure CLI:
    
    ```bash
    $ curl -sL https://aka.ms/InstallAzureCLIDeb | sudo bash  # Linux
    $ brew install azure-cli                                  # macOS
    ```
    
2. Login to Azure:
    
    ```bash
    $ az login
    ```
    

### **🔹 Configuring Terraform for GCP**

1. Install Google Cloud SDK:
    
    ```bash
    $ sudo apt install google-cloud-sdk -y  # Ubuntu
    $ brew install google-cloud-sdk         # macOS
    ```
    
2. Authenticate:
    
    ```bash
    $ gcloud auth application-default login
    ```
    

## **2.3 Understanding HCL (HashiCorp Configuration Language)**

Terraform uses **HCL (HashiCorp Configuration Language)** to define infrastructure.

### **Basic Structure of Terraform Configuration**

```hcl
provider "aws" {       # Define the cloud provider
  region = "us-east-1" # Specify the region
}

resource "aws_instance" "example" {  # Define an AWS EC2 instance
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
}
```

### **HCL Features**

- **Declarative**: You describe what infrastructure should exist, and Terraform makes it happen.
- **Modular**: Infrastructure components are reusable.
- **Human-readable**: Simpler than JSON/YAML.

## **2.4 Terraform CLI Basics**

Terraform uses a **set of CLI commands** to manage infrastructure.

| Command | Description |
| --- | --- |
| `terraform init` | Initializes a new or existing Terraform project |
| `terraform plan` | Shows what Terraform will change before applying |
| `terraform apply` | Creates or updates infrastructure |
| `terraform destroy` | Destroys all resources |
| `terraform fmt` | Formats the configuration files |
| `terraform validate` | Validates syntax errors in Terraform code |

## **Example: Install Terraform and Create a Simple EC2 Instance on AWS**

### **Step 1: Create a Terraform Project**

1. Create a new directory for your project:
    
    ```bash
    $ mkdir terraform-ec2 && cd terraform-ec2
    ```
    
2. Create a file named **`main.tf`**:
    
    ```bash
    $ touch main.tf
    ```
    
3. Open `main.tf` and add the following configuration:
    
    ```hcl
    provider "aws" {
      region = "us-east-1"
    }
    
    resource "aws_instance" "my_instance" {
      ami           = "ami-0c55b159cbfafe1f0" # Amazon Linux 2 AMI
      instance_type = "t2.micro"
    
      tags = {
        Name = "MyTerraformInstance"
      }
    }
    ```
    

---

### **Step 2: Initialize Terraform**

Before using Terraform, we need to initialize the working directory:

```bash
$ terraform init
```

📌 **What happens?**

- Downloads the AWS provider plugin.
- Creates a `.terraform` directory for dependencies.

### **Step 3: Plan Terraform Execution**

Before creating resources, preview the changes:

```bash
$ terraform plan
```

📌 **What happens?**

- Terraform shows what resources it will create.
- Helps verify changes before applying.

### **Step 4: Apply Terraform Configuration**

Create the EC2 instance by running:

```bash
$ terraform apply -auto-approve
```

📌 **What happens?**

- Terraform provisions an EC2 instance in AWS.
- Displays the public IP of the instance.

### **Step 5: Verify the EC2 Instance**

1. Go to **AWS Console** → **EC2 Dashboard**
2. You should see a running EC2 instance named **"MyTerraformInstance"**.
3. You can also check via CLI:
    
    ```bash
    $ aws ec2 describe-instances --filters "Name=tag:Name,Values=MyTerraformInstance"
    ```
    

### **Step 6: Destroy the Infrastructure**

To clean up and remove the EC2 instance:

```bash
$ terraform destroy -auto-approve
```

📌 **What happens?**

- Terraform deletes all resources defined in `main.tf`.
- Ensures no unnecessary costs.

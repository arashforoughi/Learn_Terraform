## **Project 1: Deploy a Highly Available Web App in AWS** 🌐

### **🔹 Overview**

- Deploy a **scalable web application** on AWS using:
✅ **EC2 Instances**✅ **Application Load Balancer (ALB)**✅ **Auto Scaling Group**✅ **RDS Database**

### **🔹 Step 1: Define AWS Provider**

```hcl
provider "aws" {
  region = "us-east-1"
}
```

### **🔹 Step 2: Create an Auto Scaling Group and Load Balancer**

```hcl
resource "aws_launch_template" "web" {
  name_prefix   = "web-template"
  image_id      = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
}

resource "aws_autoscaling_group" "web" {
  vpc_zone_identifier = ["subnet-xxxxx"]
  desired_capacity    = 2
  max_size           = 5
  min_size           = 1
  launch_template {
    id      = aws_launch_template.web.id
    version = "$Latest"
  }
}

resource "aws_lb" "web_lb" {
  name               = "web-alb"
  internal           = false
  load_balancer_type = "application"
  security_groups    = ["sg-xxxxx"]
  subnets           = ["subnet-xxxxx"]
}
```

✅ **Auto Scaling ensures high availability!**

### **🔹 Step 3: Deploy a Database (RDS)**

```hcl
resource "aws_db_instance" "web_db" {
  allocated_storage    = 20
  engine              = "mysql"
  instance_class      = "db.t2.micro"
  username           = "admin"
  password           = "password123"
}
```

✅ **Separate database layer for better security & scaling!**

    }

## **Project 3: Automate Infrastructure Provisioning in a CI/CD Pipeline** 🤖

### **🔹 Overview**

- **Integrate Terraform with GitLab CI/CD**
- Automate infrastructure deployment on AWS

### **🔹 Step 1: Define GitLab CI/CD Pipeline**

Create `.gitlab-ci.yml`:

```yaml
stages:
  - plan
  - apply

terraform:
  image: hashicorp/terraform:latest
  script:
    - terraform init
    - terraform plan -out=tfplan
  only:
    - main

terraform_apply:
  image: hashicorp/terraform:latest
  script:
    - terraform apply -auto-approve tfplan
  only:
    - main
```

✅ **GitLab CI/CD automates Terraform deployments!**

### **🛠 Overview**

A **Bastion Host** is a secure **jump server** that provides access to private instances within a VPC.

### **✅ Key Components**

- **VPC** with public and private subnets
- **Security Groups** for controlled SSH access
- **Bastion Host** (EC2 instance) in the public subnet
- **Private EC2 Instance** accessible only via the Bastion Host

## **Step 1: Define AWS Provider**

```hcl
provider "aws" {
  region = "us-east-1"
}
```

## **Step 2: Create a VPC & Subnets**

```hcl
resource "aws_vpc" "main_vpc" {
  cidr_block = "10.0.0.0/16"
}

resource "aws_subnet" "public_subnet" {
  vpc_id                  = aws_vpc.main_vpc.id
  cidr_block              = "10.0.1.0/24"
  map_public_ip_on_launch = true
}

resource "aws_subnet" "private_subnet" {
  vpc_id     = aws_vpc.main_vpc.id
  cidr_block = "10.0.2.0/24"
}
```

✅ **Created a VPC with public & private subnets!**

## **Step 3: Create a Security Group for the Bastion Host**

```hcl
resource "aws_security_group" "bastion_sg" {
  vpc_id = aws_vpc.main_vpc.id

  ingress {
    from_port   = 22
    to_port     = 22
    protocol    = "tcp"
    cidr_blocks = ["YOUR_IP/32"]  # Restrict access to your IP
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
  }
}
```

✅ **Only allows SSH from your IP!**

## **Step 4: Deploy the Bastion Host in the Public Subnet**

```hcl
resource "aws_instance" "bastion" {
  ami             = "ami-0c55b159cbfafe1f0"  # Update with the latest AMI
  instance_type   = "t2.micro"
  subnet_id       = aws_subnet.public_subnet.id
  security_groups = [aws_security_group.bastion_sg.name]

  tags = {
    Name = "BastionHost"
  }
}
```

✅ **Bastion Host is deployed!**

## **Step 5: Create a Private EC2 Instance (Only Accessible via Bastion)**

```hcl
resource "aws_security_group" "private_sg" {
  vpc_id = aws_vpc.main_vpc.id

  ingress {
    from_port   = 22
    to_port     = 22
    protocol    = "tcp"
    security_groups = [aws_security_group.bastion_sg.id]  # Allow SSH only from Bastion
  }
}

resource "aws_instance" "private_instance" {
  ami             = "ami-0c55b159cbfafe1f0"
  instance_type   = "t2.micro"
  subnet_id       = aws_subnet.private_subnet.id
  security_groups = [aws_security_group.private_sg.name]

  tags = {
    Name = "PrivateInstance"
  }
}
```

✅ **The private instance is only accessible through the Bastion Host!**

## **Step 6: Connect to the Private EC2 via Bastion Host**

1️⃣ **SSH into the Bastion Host**

```
$ ssh -i your-key.pem ec2-user@<bastion-public-ip>
```

2️⃣ **SSH into the Private Instance from the Bastion Host**

```
$ ssh -i your-key.pem ec2-user@10.0.2.X
```

✅ **Now, you can securely access private instances!**

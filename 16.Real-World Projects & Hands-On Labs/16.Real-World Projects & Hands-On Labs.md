## **Project 1: Deploy a Highly Available Web App in AWS** 🌐

### **🔹 Overview**

- Deploy a **scalable web application** on AWS using:
✅ **EC2 Instances**✅ **Application Load Balancer (ALB)**✅ **Auto Scaling Group**✅ **RDS Database**

### **🔹 Step 1: Define AWS Provider**

```hcl
provider "aws" {
  region = "us-east-1"
}
```

### **🔹 Step 2: Create an Auto Scaling Group and Load Balancer**

```hcl
resource "aws_launch_template" "web" {
  name_prefix   = "web-template"
  image_id      = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
}

resource "aws_autoscaling_group" "web" {
  vpc_zone_identifier = ["subnet-xxxxx"]
  desired_capacity    = 2
  max_size           = 5
  min_size           = 1
  launch_template {
    id      = aws_launch_template.web.id
    version = "$Latest"
  }
}

resource "aws_lb" "web_lb" {
  name               = "web-alb"
  internal           = false
  load_balancer_type = "application"
  security_groups    = ["sg-xxxxx"]
  subnets           = ["subnet-xxxxx"]
}
```

✅ **Auto Scaling ensures high availability!**

### **🔹 Step 3: Deploy a Database (RDS)**

```hcl
resource "aws_db_instance" "web_db" {
  allocated_storage    = 20
  engine              = "mysql"
  instance_class      = "db.t2.micro"
  username           = "admin"
  password           = "password123"
}
```

✅ **Separate database layer for better security & scaling!**

## **Project 2: Set Up a Kubernetes Cluster with Terraform** ☸️

### **🔹 Overview**

- Use Terraform to **provision a Kubernetes cluster on AWS (EKS)**
- Deploy a sample **Nginx application**

### **🔹 Step 1: Deploy an EKS Cluster**

```hcl
module "eks" {
  source          = "terraform-aws-modules/eks/aws"
  cluster_name    = "my-cluster"
  cluster_version = "1.27"
  subnets         = ["subnet-xxxxx"]
  vpc_id          = "vpc-xxxxx"
}
```

### **🔹 Step 2: Deploy an Nginx Application**

```hcl
resource "kubernetes_deployment" "nginx" {
  metadata {
    name = "nginx"
  }
  spec {
    replicas = 2
    template {
      metadata {
        labels = {
          app = "nginx"
        }
      }
      spec {
        container {
          image = "nginx:latest"
          name  = "nginx"
        }
      }
    }
  }
}
```

✅ **Deploying apps with Terraform on Kubernetes!**

## **Project 3: Automate Infrastructure Provisioning in a CI/CD Pipeline** 🤖

### **🔹 Overview**

- **Integrate Terraform with GitLab CI/CD**
- Automate infrastructure deployment on AWS

### **🔹 Step 1: Define GitLab CI/CD Pipeline**

Create `.gitlab-ci.yml`:

```yaml
stages:
  - plan
  - apply

terraform:
  image: hashicorp/terraform:latest
  script:
    - terraform init
    - terraform plan -out=tfplan
  only:
    - main

terraform_apply:
  image: hashicorp/terraform:latest
  script:
    - terraform apply -auto-approve tfplan
  only:
    - main
```

✅ **GitLab CI/CD automates Terraform deployments!**

## **Project 4: Multi-Cloud Setup (Deploy the same infrastructure in AWS & Azure)** 🌍

### **🔹 Overview**

- **Deploy identical infrastructure on AWS & Azure**
- **Ensure redundancy and failover**

### **🔹 Step 1: Define Multi-Cloud Providers**

```hcl
provider "aws" {
  region = "us-east-1"
}

provider "azurerm" {
  features {}
}
```

### **🔹 Step 2: Deploy Virtual Machines on Both Clouds**

```hcl
resource "aws_instance" "web_aws" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
}

resource "azurerm_virtual_machine" "web_azure" {
  name                = "web-vm"
  location            = "East US"
  resource_group_name = "my-resource-group"
  vm_size             = "Standard_B1s"
}
```

✅ **Same infrastructure deployed across AWS & Azure!**

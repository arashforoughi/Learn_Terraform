# 15. Advanced Terraform Topics

This chapter explores advanced Terraform concepts, including **multi-cloud deployments, writing custom providers, dynamic data sources, and performance optimizations**.

## **15.1 Managing Multi-Cloud Deployments** 🌍

Multi-cloud deployments allow infrastructure to be managed **across multiple cloud providers (AWS, Azure, GCP, etc.)** using a single Terraform configuration.

### **🔹 Step 1: Define Multi-Cloud Providers in Terraform**

Modify `providers.tf` to include **AWS and Azure**:

```hcl
terraform {
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
    azurerm = {
      source  = "hashicorp/azurerm"
      version = "~> 3.0"
    }
  }
}

provider "aws" {
  region = "us-east-1"
}

provider "azurerm" {
  features {}
}
```

✅ **Terraform can now deploy resources in AWS and Azure!**

### **🔹 Step 2: Deploy Resources in Multiple Clouds**

📌 **Example: Deploy an EC2 instance in AWS and a Virtual Machine in Azure**

```hcl
resource "aws_instance" "web" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
}

resource "azurerm_virtual_machine" "vm" {
  name                = "azure-vm"
  location            = "East US"
  resource_group_name = "my-resource-group"
  vm_size             = "Standard_B1s"
  network_interface_ids = [azurerm_network_interface.vm.id]

  storage_os_disk {
    name              = "osdisk"
    caching           = "ReadWrite"
    create_option     = "FromImage"
    managed_disk_type = "Standard_LRS"
  }
}
```

✅ **Terraform applies this configuration across AWS and Azure!**

## **15.2 Writing Custom Terraform Providers** 🛠️

Terraform allows you to create **custom providers** if a service **is not officially supported**.

### **🔹 Step 1: Install the Terraform Plugin SDK**

```bash
$ go install github.com/hashicorp/terraform-plugin-sdk/v2
```

### **🔹 Step 2: Create a Custom Provider in Go**

📌 **Example: Custom Provider to Fetch Weather Data**

```go
package main

import (
    "github.com/hashicorp/terraform-plugin-sdk/v2/helper/schema"
)

func Provider() *schema.Provider {
    return &schema.Provider{
        ResourcesMap: map[string]*schema.Resource{
            "weather_temperature": resourceWeatherTemperature(),
        },
    }
}
```

✅ **This allows Terraform to fetch weather data dynamically!**

## **15.3 Using Dynamic Data Sources** 🔄

Terraform can pull **external data** dynamically using the `external` data source.

📌 **Example: Fetch an external API response and use it in Terraform**

```hcl
data "external" "example" {
  program = ["python3", "${path.module}/fetch_data.py"]
}

output "api_data" {
  value = data.external.example.result
}
```

📌 **Python Script (`fetch_data.py`)**

```python
import json

print(json.dumps({"result": "Hello from API!"}))
```

✅ **Terraform dynamically pulls data from an external script!**

## **15.4 Performance Optimization** ⚡

Terraform performance can be optimized using:

✅ **Parallel Execution**: Run Terraform in parallel for faster execution

✅ **State Management**: Use remote state with locking (e.g., S3 + DynamoDB)

✅ **Module Caching**: Cache frequently used modules for faster deployment

📌 **Example: Enable Parallelism in Terraform**

```bash
$ terraform apply -parallelism=10
```

✅ **Faster deployments by running multiple tasks in parallel!**

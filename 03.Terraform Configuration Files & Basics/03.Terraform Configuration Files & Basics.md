# 3. Terraform Configuration Files & Basics

Now that we have Terraform installed and have deployed an EC2 instance, let's dive deeper into Terraform **configuration files**, **project structure**, and **core components** like **providers, resources, variables, and outputs**.

## **3.1 Structure of a Terraform Project**

A Terraform project typically consists of several files:

| File Name | Description |
| --- | --- |
| `main.tf` | Defines the infrastructure (resources) |
| `variables.tf` | Stores input variables |
| `outputs.tf` | Defines output values |
| `terraform.tfvars` | Stores variable values (optional) |
| `providers.tf` | Defines the provider configuration (optional) |
| `terraform.tfstate` | Stores the current state of the infrastructure |
| `.terraform/` | Directory containing Terraform provider plugins |

📌 **Example Project Structure**

```css
terraform-project/
│── main.tf
│── variables.tf
│── outputs.tf
│── terraform.tfvars
│── .terraform/
│── terraform.tfstate
```

## **3.2 Writing Basic Terraform Configuration**

Every Terraform configuration must specify:

1. **Provider**: Defines the cloud provider (e.g., AWS, Azure, GCP).
2. **Resources**: The infrastructure components (e.g., EC2, S3, RDS).
3. **Variables**: Dynamic input values.
4. **Outputs**: Display important information.

### **Basic Example: Creating an AWS S3 Bucket**

```hcl
provider "aws" {
  region = "us-east-1"
}

resource "aws_s3_bucket" "my_bucket" {
  bucket = "my-terraform-bucket-12345"
}
```

📌 **Commands to Apply**

```bash
$ terraform init
$ terraform apply -auto-approve
```

➡ This creates an **S3 bucket** in AWS.

## **3.3 Providers, Resources, and Data Sources**

### **🔹 Providers**

A provider is a plugin that allows Terraform to interact with external services.

Example: **Defining AWS as a provider**

```hcl
provider "aws" {
  region = "us-east-1"
}
```

Terraform supports many providers like **AWS, Azure, GCP, Kubernetes, Vault, GitHub, etc.**

➡ **To view all available providers:** Terraform Registry

### **🔹 Resources**

A **resource** is any infrastructure component managed by Terraform (e.g., EC2, S3, RDS).

Example: **Creating an EC2 instance**

```hcl
resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
}
```

### **🔹 Data Sources**

A **data source** allows Terraform to **fetch existing information** from a provider.

Example: **Fetching the latest Amazon Linux 2 AMI**

```hcl
data "aws_ami" "latest_amazon_linux" {
  most_recent = true
  owners      = ["amazon"]
  filter {
    name   = "name"
    values = ["amzn2-ami-hvm-*"]
  }
}
```

📌 **Usage in an EC2 instance**

```hcl
resource "aws_instance" "example" {
  ami           = data.aws_ami.latest_amazon_linux.id
  instance_type = "t2.micro"
}
```

➡ Terraform dynamically fetches the latest AMI.

## **3.4 Variables and Outputs**

### **🔹 Variables**

Variables allow us to **parametrize** Terraform configurations.

Example: **Defining a variable in `variables.tf`**

```hcl
variable "bucket_name" {
  description = "The name of the S3 bucket"
  type        = string
  default     = "my-default-bucket"
}
```

📌 **Using the variable in `main.tf`**

```hcl
resource "aws_s3_bucket" "my_bucket" {
  bucket = var.bucket_name
}
```

📌 **Passing a variable during `terraform apply`**

```bash
$ terraform apply -var="bucket_name=my-custom-bucket"
```

### **🔹 Outputs**

Outputs allow Terraform to return important information.

Example: **Defining outputs in `outputs.tf`**

```hcl
output "s3_bucket_name" {
  description = "The name of the created S3 bucket"
  value       = aws_s3_bucket.my_bucket.id
}
```

📌 **Displaying outputs**

```bash
$ terraform output
```

➡ Shows the bucket name in the terminal.

## **3.5 Dependencies in Terraform**

Terraform automatically manages dependencies based on resource relationships.

Example: **EC2 instance depends on a security group**

```hcl
resource "aws_security_group" "my_sg" {
  name = "allow_http"
  ingress {
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }
}

resource "aws_instance" "my_instance" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"
  security_groups = [aws_security_group.my_sg.name]
}
```

➡ Terraform ensures the security group is created **before** the EC2 instance.

## **Example: Deploy an AWS S3 Bucket with Variables and Outputs**

### **Step 1: Create a Terraform Project**

```bash
$ mkdir terraform-s3 && cd terraform-s3
```

### **Step 2: Create `variables.tf`**

```hcl
variable "bucket_name" {
  description = "Name of the S3 bucket"
  type        = string
}

variable "region" {
  description = "AWS region"
  type        = string
  default     = "us-east-1"
}
```

### **Step 3: Create `main.tf`**

```hcl
provider "aws" {
  region = var.region
}

resource "aws_s3_bucket" "my_bucket" {
  bucket = var.bucket_name
}
```

### **Step 4: Create `outputs.tf`**

```hcl
output "bucket_id" {
  description = "The ID of the created S3 bucket"
  value       = aws_s3_bucket.my_bucket.id
}
```

### **Step 5: Apply Terraform**

```bash
$ terraform init
$ terraform apply -var="bucket_name=my-terraform-bucket-123" -auto-approve
```

➡ Terraform creates an **S3 bucket** and prints the bucket name.

### **Step 6: Destroy the Infrastructure**

```bash
$ terraform destroy -auto-approve
```

➡ Deletes the **S3 bucket** to avoid extra costs.

# 10. Terraform Loops & Conditionals

Terraform provides looping and conditional mechanisms to create **dynamic configurations** efficiently. Instead of writing repetitive code, we can use **count, for_each, and dynamic blocks** to simplify our infrastructure as code.

## **10.1 count vs. for_each**

Terraform offers two looping mechanisms:

| Feature | `count` | `for_each` |
| --- | --- | --- |
| **Type** | Uses an integer for iteration | Uses a map or set for iteration |
| **Best for** | Creating multiple similar resources | Creating resources dynamically with unique values |
| **Key-based indexing** | No (uses index numbers) | Yes (uses keys from a map or set) |
| **Removing an item** | Can cause reordering issues | No reordering issues |

### **🔹 Using `count`**

📌 **Example: Create 3 EC2 Instances Using `count`**

```hcl
resource "aws_instance" "example" {
  count         = 3
  ami           = "ami-12345678"
  instance_type = "t2.micro"
  tags = {
    Name = "server-${count.index}"
  }
}
```

✅ This creates **3 instances** (`server-0`, `server-1`, `server-2`).

❌ **Issue**: If an instance is removed, Terraform might **recreate instances** to maintain order.

### **🔹 Using `for_each`**

📌 **Example: Create EC2 Instances Dynamically Using `for_each`**

```hcl
variable "instances" {
  type = map(string)
  default = {
    "web"  = "t2.micro"
    "app"  = "t3.small"
    "db"   = "t3.medium"
  }
}

resource "aws_instance" "example" {
  for_each      = var.instances
  ami           = "ami-12345678"
  instance_type = each.value
  tags = {
    Name = each.key
  }
}
```

✅ This creates **3 instances with unique names (`web`, `app`, `db`)**.

✅ **Removing an instance** (e.g., `"db"`) does **not** affect others.

## **10.2 Dynamic Blocks**

Dynamic blocks **reduce repetition** when defining nested configurations.

📌 **Example: Create Security Groups with Dynamic Ingress Rules**

```hcl
variable "ingress_rules" {
  default = [
    { port = 80,  protocol = "tcp" },
    { port = 443, protocol = "tcp" }
  ]
}

resource "aws_security_group" "example" {
  name = "dynamic-sg"

  dynamic "ingress" {
    for_each = var.ingress_rules
    content {
      from_port   = ingress.value.port
      to_port     = ingress.value.port
      protocol    = ingress.value.protocol
      cidr_blocks = ["0.0.0.0/0"]
    }
  }
}
```

✅ This **dynamically adds ingress rules** for multiple ports **without repetition**.

## **10.3 Using Conditional Expressions**

Terraform allows conditionals using the **ternary operator (`? :`)**.

📌 **Example: Assigning Instance Type Based on Environment**

```hcl
variable "environment" {
  default = "dev"
}

resource "aws_instance" "example" {
  ami           = "ami-12345678"
  instance_type = var.environment == "prod" ? "t3.large" : "t2.micro"
}
```

✅ If `environment = "prod"`, it assigns `"t3.large"`, else `"t2.micro"`.

## **Scenario: Creating Multiple EC2 Instances Dynamically Using `for_each`**

📌 **Step 1: Define a Variable for EC2 Instances**

```hcl
variable "instances" {
  type = map(object({
    instance_type = string
    volume_size   = number
  }))
  default = {
    "web"  = { instance_type = "t2.micro", volume_size = 10 }
    "app"  = { instance_type = "t3.small", volume_size = 20 }
  }
}
```

📌 **Step 2: Define the EC2 Resource with `for_each`**

```hcl
resource "aws_instance" "example" {
  for_each      = var.instances
  ami           = "ami-12345678"
  instance_type = each.value.instance_type

  root_block_device {
    volume_size = each.value.volume_size
  }

  tags = {
    Name = each.key
  }
}
```

✅ This creates **two EC2 instances**:

- `web` → `t2.micro`, **10GB** volume
- `app` → `t3.small`, **20GB** volume

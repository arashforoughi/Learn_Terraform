# 14. Terraform Cloud & Enterprise

Terraform Cloud and Terraform Enterprise provide **collaborative infrastructure management**, **remote state storage**, and **policy enforcement** with Sentinel. This chapter covers **Terraform Cloud basics, remote state storage, and using Sentinel to enforce security policies**.

## **14.1 Overview of Terraform Cloud**

### **🔹 What is Terraform Cloud?**

Terraform Cloud is a **managed service by HashiCorp** that enables:

✅ **Remote execution** of Terraform runs

✅ **State management and storage**

✅ **Team collaboration with role-based access control (RBAC)**

✅ **Policy enforcement using Sentinel**

📌 **Terraform Cloud vs. Self-Hosted Terraform**

| Feature | Terraform Cloud ☁️ | Self-Hosted Terraform 🏠 |
| --- | --- | --- |
| State Management | Built-in (SaaS) | Manual (e.g., S3, Git) |
| Execution Environments | Cloud Runners | Local/CI Pipeline |
| Security & Compliance | Sentinel Policies | Manual Implementation |
| Collaboration | RBAC, Notifications | GitOps, CI/CD Pipelines |

✅ **Terraform Cloud simplifies team-based infrastructure management**.

## **14.2 Remote State Storage in Terraform Cloud**

By default, Terraform stores state files **locally** (`terraform.tfstate`). **Terraform Cloud provides a centralized backend** for state storage.

### **🔹 Step 1: Create a Terraform Cloud Account**

1. Sign up at app.terraform.io
2. Create a **new organization**
3. Create a **workspace** for managing infrastructure

### **🔹 Step 2: Configure Terraform to Use Terraform Cloud**

Modify `main.tf` to use **Terraform Cloud as the backend**:

```hcl
terraform {
  cloud {
    organization = "my-terraform-org"

    workspaces {
      name = "my-terraform-workspace"
    }
  }
}
```

Then, **initialize Terraform**:

```bash
$ terraform init
```

✅ **Terraform will authenticate with Terraform Cloud and store state remotely.**

### **🔹 Step 3: Run Terraform in Terraform Cloud**

After initialization, **execute Terraform commands** in Terraform Cloud:

```bash
$ terraform plan
$ terraform apply
```

Alternatively, you can **trigger a Terraform run from the Terraform Cloud UI**.

## **14.3 Using Sentinel for Policy Enforcement**

Sentinel is a **policy-as-code framework** that enforces security rules **before Terraform applies changes**.

### **🔹 Step 1: Enable Sentinel Policies**

In Terraform Cloud:

1. Navigate to **Settings → Policy Sets**
2. Create a **new policy set**
3. Link the policy set to your **workspace**

### **🔹 Step 2: Define a Sentinel Policy**

📌 **Example: Restrict EC2 Instances to Specific Regions**

Create `restrict-region.sentinel`:

```hcl
import "tfplan/v2" as tfplan

allowed_regions = ["us-east-1", "us-west-2"]

main = rule {
  all tfplan.resources.aws_instance as _, instances {
    all instances.applied.resource_changes as instance {
      instance.change.after.region in allowed_regions
    }
  }
}
```

✅ **Terraform will block deployments outside `us-east-1` and `us-west-2`.**

### **🔹 Step 3: Test the Sentinel Policy**

Run a **policy check**:

```bash
$ terraform apply
```

If an **EC2 instance is created in an unauthorized region**, Terraform **blocks** the change:

```bash
Policy Violation: Resource 'aws_instance.web' is in an unauthorized region 'eu-central-1'.
```

✅ **Ensures compliance with security policies.**

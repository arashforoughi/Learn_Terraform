# 5. Terraform Variables & Outputs

Terraform variables help make configurations **dynamic, reusable, and flexible**. In this chapter, we'll explore **different variable types, input and output variables, and handling sensitive data**.

## **5.1 Types of Variables in Terraform**

Terraform supports several types of variables:

✔ **String** - Stores a single string value

✔ **Number** - Stores numeric values

✔ **Boolean** - Stores `true` or `false`

✔ **List** - Stores an array of values

✔ **Map** - Stores key-value pairs

✔ **Object** - Stores structured data

✔ **Tuple** - Stores mixed data types

📌 **Defining Different Variable Types (`variables.tf`)**

```hcl
variable "instance_type" {
  description = "Type of EC2 instance"
  type        = string
  default     = "t2.micro"
}

variable "port_numbers" {
  description = "List of ports to allow in security group"
  type        = list(number)
  default     = [80, 443, 22]
}

variable "instance_tags" {
  description = "Tags for EC2 instance"
  type        = map(string)
  default = {
    Name  = "WebServer"
    Env   = "staging"
    Owner = "DevOps"
  }
}
```

📌 **Using the variables in `main.tf`**

```hcl
resource "aws_instance" "example" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = var.instance_type
  tags          = var.instance_tags
}
```

📌 **Passing variable values during `terraform apply`**

```bash
$ terraform apply -var="instance_type=t3.medium"
```

## **5.2 Using Input Variables**

### **🔹 Default & Override Values**

If a variable has a **default value**, it is optional during `terraform apply`.

If no default is set, Terraform requires an explicit value.

📌 **Overriding values using `terraform.tfvars`**
Create a `terraform.tfvars` file:

```hcl
instance_type = "t3.medium"
```

Run:

```bash
$ terraform apply -var-file="terraform.tfvars"
```

## **5.3 Using Output Variables**

Output variables allow Terraform to **return important resource details**.

📌 **Defining an output (`outputs.tf`)**

```hcl
output "instance_public_ip" {
  description = "Public IP of the instance"
  value       = aws_instance.example.public_ip
}
```

📌 **Fetching outputs after deployment**

```bash
$ terraform output
```

## **5.4 Sensitive Variables (Secrets Management)**

By default, Terraform **displays variable values in logs**, which can expose secrets.

To prevent this, mark variables as **sensitive**.

📌 **Handling Sensitive Data (`variables.tf`)**

```hcl
variable "db_password" {
  description = "Database password"
  type        = string
  sensitive   = true
}
```

📌 **Using the sensitive variable in `main.tf`**

```hcl
resource "aws_db_instance" "example" {
  engine         = "mysql"
  instance_class = "db.t3.micro"
  password       = var.db_password
}
```

📌 **Hiding Sensitive Outputs (`outputs.tf`)**

```hcl
output "database_password" {
  value     = var.db_password
  sensitive = true
}
```

➡ Terraform **hides** sensitive values in the CLI output.

## **Example: Create a VPC with Dynamic CIDR Block**

We'll create a **VPC** where the **CIDR block is dynamically assigned via variables**.

### **🔹 Step 1: Define Variables (`variables.tf`)**

```hcl
variable "vpc_cidr" {
  description = "CIDR block for the VPC"
  type        = string
}

variable "environment" {
  description = "Deployment environment"
  type        = string
  default     = "staging"
}
```

### **🔹 Step 2: Create VPC (`main.tf`)**

```hcl
provider "aws" {
  region = "us-east-1"
}

resource "aws_vpc" "my_vpc" {
  cidr_block = var.vpc_cidr

  tags = {
    Name        = "MyVPC-${var.environment}"
    Environment = var.environment
  }
}
```

### **🔹 Step 3: Define Outputs (`outputs.tf`)**

```hcl
output "vpc_id" {
  description = "The ID of the created VPC"
  value       = aws_vpc.my_vpc.id
}
```

### **🔹 Step 4: Apply Terraform with Variables**

Run:

```bash
$ terraform init
$ terraform apply -var="vpc_cidr=10.0.0.0/16" -auto-approve
```

➡ Terraform **dynamically assigns** the CIDR block.

### **🔹 Step 5: Retrieve VPC ID**

Run:

```bash
$ terraform output vpc_id
```

➡ Terraform **displays the VPC ID**.

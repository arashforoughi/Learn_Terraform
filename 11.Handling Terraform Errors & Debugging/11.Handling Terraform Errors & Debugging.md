# 11. Handling Terraform Errors & Debugging 

Terraform provides various tools to handle errors and debug issues efficiently. Understanding **error messages**, using **debug logs**, and applying **Terraform commands like `fmt`, `validate`, `taint`, and `refresh`** can help in troubleshooting problems.

## **11.1 Understanding Terraform Errors**

Terraform errors typically fall into these categories:

| Error Type | Description |
| --- | --- |
| **Syntax Errors** | Incorrect HCL syntax (e.g., missing `{}` or `=`) |
| **Validation Errors** | Invalid resource names, unsupported arguments, or incorrect values |
| **Dependency Errors** | Errors due to missing dependencies between resources |
| **Provider Errors** | Issues in cloud provider authentication or configuration |
| **State Errors** | Corrupt or outdated state file issues |

### **🔹 Example: Common Terraform Error (Syntax Error)**

❌ **Incorrect Code (missing `=` in `instance_type`)**

```hcl
resource "aws_instance" "example" {
  ami "ami-12345678"
  instance_type "t2.micro"
}
```

✅ **Fix: Add `=`**

```hcl
resource "aws_instance" "example" {
  ami           = "ami-12345678"
  instance_type = "t2.micro"
}
```

➡ Terraform will now **parse the file correctly**.

## **11.2 Debugging with Terraform Logs**

Terraform provides logging levels for debugging:

| Log Level | Purpose |
| --- | --- |
| `TRACE` | Most detailed logs (low-level Terraform internals) |
| `DEBUG` | Debug-level logs (useful for diagnosing problems) |
| `INFO` | Standard logs (shows major steps) |
| `WARN` | Warnings (non-critical issues) |
| `ERROR` | Errors (critical failures) |

### **🔹 Enabling Debug Logs**

```bash
$ export TF_LOG=DEBUG
$ terraform apply
```

➡ Terraform will print detailed debug logs.

### **🔹 Saving Debug Logs to a File**

```bash
$ export TF_LOG=DEBUG
$ export TF_LOG_PATH="terraform-debug.log"
$ terraform apply
```

➡ All logs are **saved in `terraform-debug.log`**.

## **11.3 Using `terraform fmt`, `validate`, `taint`, `refresh`**

### **🔹 `terraform fmt` (Fix Formatting Issues)**

Automatically formats Terraform files according to best practices.

📌 **Example: Format All Terraform Files**

```
$ terraform fmt
```

✅ Removes unnecessary spaces and fixes indentation.

### **🔹 `terraform validate` (Validate Configuration)**

Checks for syntax or logical errors **without executing any changes**.

📌 **Example: Validate a Configuration**

```
$ terraform validate
```

✅ Returns errors if there are **incorrect resource names, missing values, etc.**

### **🔹 `terraform taint` (Force Resource Recreation)**

Marks a resource as **tainted**, forcing Terraform to **recreate it** in the next `apply`.

📌 **Example: Force Recreate an EC2 Instance**

```
$ terraform taint aws_instance.example
$ terraform apply
```

✅ This **destroys and recreates** the EC2 instance.

### **🔹 `terraform refresh` (Sync State with Reality)**

Updates the **Terraform state file** with the actual **live state** of the infrastructure.

📌 **Example: Refresh Terraform State**

```
$ terraform refresh
```

✅ Useful when changes are made **manually in the cloud provider**.

## **Scenario: Troubleshoot a Terraform Configuration Where an EC2 Instance is Not Being Created Properly**

📌 **Problem: Terraform is Not Creating an EC2 Instance**

```hcl
resource "aws_instance" "example" {
  ami           = "ami-12345678"
  instance_type = var.instance_size
}
```

📌 **Step 1: Run `terraform validate` to Check for Errors**

```
$ terraform validate
```

❌ **Output:**

```vbnet
Error: Reference to undeclared input variable
"var.instance_size" not defined
```

➡ **Solution:** Declare the variable in `variables.tf`

```hcl
variable "instance_size" {
  default = "t2.micro"
}
```

📌 **Step 2: Run Terraform in Debug Mode**

```bash
$ export TF_LOG=DEBUG
$ terraform apply
```

❌ **Error Message:**

```jsx
Error: Invalid AMI ID
```

➡ **Solution:** Use a valid AMI ID.

📌 **Step 3: Force Recreate the EC2 Instance**

```bash
$ terraform taint aws_instance.example
$ terraform apply
```

✅ Terraform **recreates** the EC2 instance.

📌 **Step 4: Refresh Terraform State**

```bash
$ terraform refresh
```

✅ Syncs the **state file** with the **actual infrastructure**.
